Note: `#111` is used both as text color and as accent color. Where it's the body/heading text, use `--color-text`. Where it's a background (active fmt-btn, download-btn) or an active underline, use `--color-accent`. Distinguish by context.

**1. Add FOUC-fix inline script in `<head>`, BEFORE the `<link rel="stylesheet">` tag:**

```html
<script>
  try {
    var t = localStorage.getItem('theme-preference');
    if (t === 'dark' || t === 'light') document.documentElement.setAttribute('data-theme', t);
  } catch(e) {}
</script>
<link rel="stylesheet" href="styles.css">
```

This runs synchronously before any CSS is parsed. If a preference is saved, `data-theme` is set on `<html>` before the stylesheet loads, so the correct token values are active from the very first paint.

**1. Add `toggleTheme()` function after the existing `downloadQR()` function:**

```javascript
function toggleTheme() {
  const current = document.documentElement.getAttribute('data-theme');
  const isDark = current === 'dark' || (!current && window.matchMedia('(prefers-color-scheme: dark)').matches);
  const next = isDark ? 'light' : 'dark';
  document.documentElement.setAttribute('data-theme', next);
  document.getElementById('theme-toggle').textContent = next === 'dark' ? '☾' : '☀';
  try { localStorage.setItem('theme-preference', next); } catch(e) {}
}
```

**2. Wire the toggle button to `toggleTheme()` — add this after `toggleTheme()`:**

```javascript
document.getElementById('theme-toggle').addEventListener('click', toggleTheme);
```
