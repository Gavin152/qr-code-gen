# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Persistence.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Phase Scope Note

Phase 2 introduces no new UI components, screens, or visible elements. All changes are
behavioral: localStorage read/write hooks on existing event listeners and a restore pass
on `DOMContentLoaded`. The design contract for this phase documents the existing visual
system (so the executor preserves it) and specifies the two new silent interaction
patterns required by CONTEXT.md decisions D-03 and D-07.

## Interaction Contracts

These two patterns are new in Phase 2 and have no visual analog in the current codebase.

### 1. Silent localStorage failure (D-07)

- **Trigger:** `localStorage` unavailable (private browsing, storage quota exceeded,
  SecurityError)
- **Behavior:** App functions normally. No toast, no banner, no console.warn required.
  Settings are not saved; the user is not informed.
- **Visual change:** None. Zero DOM changes on failure.
- **Implementation note:** Wrap all `localStorage.setItem` and `localStorage.getItem`
  calls in a try/catch. On catch, continue execution without mutation.

### 2. Logo save size cap (D-03)

- **Trigger:** Base64 logo string length exceeds 500,000 characters (~375KB raw,
  ~500KB base64) at time of logo upload.
- **Behavior:** Skip saving the logo image to localStorage. Save the logo size value
  (`logoSize`) as normal. Continue silently — no error shown to the user, no visual
  state change.
- **Visual change:** None. The logo still renders in the QR canvas for the current
  session; it simply will not be restored on the next page load.
- **Implementation note:** Check `logoDataUrl.length > 500000` before calling
  `localStorage.setItem` for the logo key. The size key saves unconditionally.

### 3. Logo restore — UI state (D-05, D-06)

Not a new visual pattern, but requires matching the existing shown/hidden state
exactly to avoid visual regressions:

- Restored logo: show `#logo-name` (set `textContent` to saved filename or a
  placeholder like "logo.png"), show `#logo-clear` button, show `#logo-size-field`.
- No logo in storage: leave all three hidden (default state).
